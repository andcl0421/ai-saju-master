# 📂 09_roadmap.md - 프로젝트 로드맵 (Roadmap)

---

## 1. 개요 (Overview)
본 문서는 '만세력 AI 서비스'의 개발 단계별 일정과 주요 마일스톤, 그리고 정식 출시 이후의 추가 기능 고도화 계획을 정의한다. 본 로드맵은 설계 단계에서 정의된 기술 스택(FastAPI, React, PostgreSQL)을 기준으로 작성되었다.

---

## 2. 개발 단계별 마일스톤 (Milestones)

### Phase 1: 기반 설계 및 환경 구축 (Week 1) - **Completed** ✅
- [x] 서비스 기획 및 요구사항 정의 (`01_concept.md`)
- [x] 데이터베이스 모델링 및 설계 (`04_db_modeling.md`)
- [x] API 상세 설계 및 엔드포인트 정의 (`06_api_design.md`)
- [x] 백엔드 개발 환경 구축 (FastAPI, PostgreSQL, Docker)

### Phase 2: 핵심 로직 및 백엔드 개발 (Week 2-3)
- [ ] 만세력 라이브러리 연동 및 사주 간지 계산 로직 구현
- [ ] RAG 시스템 구축 (Vector DB 데이터 적재 및 검색 로직)
- [ ] LLM 프롬프트 파이프라인 완성 (`07_prompt_design.md` 기반)
- [ ] 핵심 API 엔드포인트 구현 (User, Profile, Fortune)

### Phase 3: 프론트엔드 연동 및 기능 최적화 (Week 4)

- [ ] 운세 캐싱 시스템 및 스케줄러(Push 알림) 연동 테스트
- [ ] 전체 서비스 통합 테스트 및 버그 수정

### Phase 4: 배포 및 운영 (Week 5)
- [ ] AWS(EC2/RDS) 인프라 구축 및 클라우드 배포
- [ ] 실사용자 피드백 수집 및 성능 모니터링 (AI 비용 추적)

## 3. 향후 고도화 계획 (Future Updates)

### v1.5: 사용자 경험 강화
- **소셜 로그인 확대**: 카카오, 네이버 등 소셜 계정 연동.
- **이미지 생성**: 운세 결과를 바탕으로 '오늘의 행운 카드' 이미지 생성 및 공유 기능.

### v2.0: 멀티 도메인 확장
- **AI 관상/손금 분석**: Vision 모델을 활용한 신체 특징 분석 기능 추가.
- **타로 모드 추가**: 사주 외에 타로 카드를 활용한 하이브리드 점술 서비스.

### v3.0: 플랫폼화
- **B2B API 오픈**: 타 플랫폼에 운세 API 유료 공급.
- **다국어 지원**: 글로벌 시장(영미권, 중화권) 타겟팅.

---

## 4. 위험 관리 (Risk Management)
- **API 비용 관리**: 캐싱 로직 고도화 및 GPT-4o mini 등 저비용 모델 혼합 사용.
- **데이터 정확도**: 명리학 데이터셋의 지속적인 검수 및 RAG 품질 개선.
