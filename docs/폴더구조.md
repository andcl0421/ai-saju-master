## 2. app/ — 백엔드 아키텍처 상세 (3계층 구조)

각 폴더는 독립적인 책임을 가지며, 상위 계층이 하위 계층을 호출하는 방식으로 작동합니다.

### 📂 app/api/ (Presentation Layer)
**[역할]** 외부 요청 수신 및 응답 반환 (Controller)
- **v1/auth.py**: 사용자 인증, 이메일 기반 온보딩 진입점.
- **v1/profiles.py**: 사주 프로필 CRUD 요청 처리기.
- **v1/fortunes.py**: AI 운세 생성 및 궁합 분석 요청 엔드포인트.
- **v1/router.py**: 위 라우터들을 하나로 묶어 FastAPI 앱에 등록하는 허브.

### 📂 app/services/ (Business Logic Layer)
**[역할]** 비즈니스 흐름 제어 및 도메인 로직 조합
- **fortune_manager.py**: DB 캐시 확인 → 도메인 엔진 호출 → AI 호출 → 결과 저장 프로세스 관리.
- **user_service.py**: 사용자 가입 처리 및 프로필 연동 로직.

### 📂 app/domain/ (Core Logic Layer)
**[역할]** 순수 사주 계산 알고리즘 (Infrastructure 의존성 없음)
- **saju_engine.py**: 생년월일시 → 육십갑자(간지) 변환 핵심 로직.
- **constants.py**: 천간, 지지, 절기 등 사주 계산용 상수 데이터.

### 📂 app/integrations/ (External API Layer)
**[역할]** 외부 인프라(LLM) 연동
- **openai_client.py**: OpenAI 비동기 호출 및 응답 파싱.
- **prompts.py**: 사주 페르소나 및 출력 형식을 정의한 프롬프트 템플릿.

### 📂 app/crud/ (Data Access Layer)
**[역할]** 데이터베이스 직접 조작 (Repository)
- **crud_user.py**: 유저 테이블 전용 조회/생성 함수.
- **crud_saju.py**: 사주 프로필 테이블 전용 DB 작업.
- **crud_fortune.py**: 운세 결과물 저장 및 캐시 조회 기능.

### 📂 app/db/ (Database Infra)
**[역할]** DB 연결 및 테이블 정의
- **session.py**: AsyncEngine 설정 및 get_db 세션 주입 도구.
- **models.py**: SQLAlchemy 기반 테이블 정의 (User, Profile, Fortune).
- **base.py**: 모든 모델을 통합 관리하는 Base 클래스 (Alembic 인식용).

### 📂 app/schemas/ (Data Validation)
**[역할]** 데이터 규격 검증 (Pydantic)
- **user.py / saju.py / fortune.py**: 각 도메인별 입출력(Request/Response) 모델.

### 📂 app/core/ (Config & Security)
**[역할]** 전역 설정 및 보안
- **config.py**: .env 파일을 읽어오는 환경 변수 관리 클래스.
- **security.py**: CORS 설정 및 권한 체크 유틸리티.
