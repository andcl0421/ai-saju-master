# 🔮 AI 사주 Master: OpenAI Function Calling 기반 지능형 서비스

## 1. 서비스 개요 (Service Overview)
* **서비스명**: AI 사주 Master (Saju Master Agent)
* **한 줄 정의**: 사용자의 사주 데이터를 기반으로 AI가 필요한 외부 지식과 계산 도구를 스스로 호출하여 운세를 생성하는 **On-Demand AI 운세 생성 서비스**.
* **핵심 기술**: OpenAI Native Function Calling + FastAPI + PostgreSQL
* **핵심 가치**: "고정된 답변이 아닙니다. AI가 질문에 맞는 최적의 도구를 직접 호출하여 '데이터 기반의 정밀 운세'를 생성합니다."

---

## 2. 서비스 목표 (Service Goals)
1. **자율적 도구 호출**: OpenAI Function Calling을 활용하여 LLM이 질문 의도에 따라 필요한 도구(사주 계산, 외부 데이터 조회)를 스스로 선택하고 실행하는 구조 확립.
2. **데이터 신뢰성 확보**: 사주 간지 계산 + 외부 명리학 데이터(URL) + LLM 해석을 결합한 Hybrid AI 구조 구축.
3. **효율적 비용 관리**: 동일 날짜 운세에 대해 DB 캐싱을 적용하여 불필요한 LLM 호출 및 토큰 소모 최소화.
4. **간편한 사용자 식별**: 복잡한 인증 절차 대신 **이메일 기반 사용자 식별**을 통해 유연한 데이터 관리와 빠른 접근성 제공.

---

## 3. 타겟 사용자 (Target Audience)
* **2030 MZ 세대**: AI 신기술과 전통 사주의 결합에 호기심이 많고, 시각화된 데이터(차트 등)를 선호하는 층.
* **심층 사주 분석 사용자**: 단순 결과가 아닌 에이전트가 어떤 근거(도구 호출)로 답변을 도출했는지 알고 싶은 신뢰 중심 사용자.
* **인연 분석 사용자**: 등록된 지인 프로필 데이터를 활용해 AI의 오행 분석을 통한 정밀 궁합을 확인하고 싶은 사용자.

---

## 4. 핵심 기능 (Core Features)
1. **Function Calling 기반 사주 분석**: 사용자의 생년월일시 정보를 바탕으로 LLM이 만세력 함수를 직접 호출하여 사주 원국 분석.
2. **실시간 외부 URL 참조**: 특정 질문 발생 시 LLM이 **외부 URL 도구를 호출**하여 최신 사주 로직 및 데이터를 답변에 반영.
3. **AI 인연 궁합 분석**: 여러 프로필 데이터를 대조하여 오행의 조화와 상생·상극을 분석하는 지능형 궁합 서비스.
4. **이메일 기반 멀티 프로필 관리**: 하나의 이메일 계정으로 본인 및 주변인의 사주 정보를 안전하게 관리하는 개인 저장소 기능.
5. **데이터 시각화 리포트**: 도출된 운세 점수(재물/연애/건강 등)를 한눈에 볼 수 있는 레이더 차트 형태의 결과 제공.
6. **On-Demand 운세 생성 및 캐싱**: 요청 시에만 AI 운세를 생성하며, 동일 날짜 요청은 DB에서 즉시 재사용하여 성능 최적화.

---

## 5. 서비스 프로세스 (Execution Workflow)
1. **사용자 식별**: 사용자가 이메일 입력 → `users` 테이블 조회 후 기존 회원 식별 또는 신규 생성.
2. **프로필 관리**: 생년월일시 정보 입력 → 서버에서 사주 간지 계산 후 `saju_profiles`에 저장.
3. **운세 요청**: 사용자가 특정 날짜의 운세 조회를 요청.
4. **캐싱 확인**: `daily_fortunes` 테이블에서 해당 프로필과 날짜의 데이터 존재 여부 확인.
5. **분기 처리**:
   - **데이터 존재**: DB에 저장된 기존 결과를 즉시 반환.
   - **데이터 없음**: AI 운세 생성 프로세스(Function Calling) 가동.
6. **AI 분석 (Action)**: 사주 간지 + 오늘 일진 + 외부 URL 데이터를 결합하여 LLM이 정밀 분석 수행.
7. **DB 저장 및 응답**: 생성된 JSON 결과를 캐싱 테이블에 저장한 후 사용자에게 최종 응답 제공.

---

## 6. 시스템 아키텍처 (Architecture)
* **Frontend**: React (사용자 입력 및 결과 시각화)
* **Backend**: FastAPI (비즈니스 로직 처리 및 API 엔드포인트 제공)
* **Database**: PostgreSQL (사용자 이메일 기반 계정, 프로필 및 운세 캐시 저장)
* **AI Engine**: OpenAI GPT-4o (Native Function Calling을 통한 자율 도구 호출)

---

## 7. 서비스 특징 (Key Characteristics)
1. **On-Demand 구조**: 모든 데이터를 미리 생성하지 않고 요청 시에만 생성하여 불필요한 API 비용 방지.
2. **DB 캐싱 전략**: 하루 1회 생성된 운세는 DB에 저장되어 이후 재접속 시 초고속 응답 가능.
3. **Function Calling 기반 Agent 구조**: AI가 수동적으로 답하는 것이 아니라 직접 필요한 도구를 선택하고 실행하는 능동적 구조.
4. **Hybrid AI 구조**: DB의 정적 데이터와 RAG(지식 검색), 그리고 LLM의 추론력을 결합한 고도화된 모델.
5. **확장성**: 향후 타로, 관상, 인공지능 작명 등 다양한 글로벌 역술 서비스로 확장 가능한 모듈형 설계.

---

## 8. 수익 모델 (Business Model)
1. **프리미엄 상세 사주 리포트**: 복합적인 도구 호출이 필요한 심층 분석 리포트 유료화.
2. **고급 궁합 분석 기능**: 다수 인원 간의 관계성 및 그룹 궁합 기능 제공.
3. **광고 기반 수익**: 효율적인 아키텍처로 유지비를 낮추고 결과 페이지 내 맞춤형 광고 배치.
4. **데이터 백업 및 연동 기능**: 이메일 기반 계정을 활용한 기기 간 데이터 동기화 및 복구 서비스.

## 10. 데이터 구조 개요 (Data Structure Overview)

서비스의 데이터 구조는 다음과 같은 계층 구조를 가진다.

User (이메일 계정)
    ↓
Saju Profiles (본인 및 지인 사주 정보)
    ↓
Daily Fortunes (날짜별 운세 캐시 데이터)

### 데이터 흐름
1. 사용자는 이메일로 계정을 생성한다.
2. 하나의 사용자 계정은 여러 개의 사주 프로필을 가질 수 있다.
3. 각 사주 프로필은 날짜별 운세 데이터를 생성하고 저장한다.
4. 동일 날짜의 운세는 재생성하지 않고 DB에서 재사용한다.

이 구조를 통해 API 비용을 절감하고 응답 속도를 향상시킨다.
